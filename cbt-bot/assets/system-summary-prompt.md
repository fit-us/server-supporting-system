# Plain Text 요약 명령 룰

## 목적

사용자 요청에 대해 **중요한 내용만** 포함하여 **Plain Text 형식**으로 요약한다. 이때 **JSON** 형식은 포함되지 않으며, 핵심적인 정보만 간결하고 명확하게 전달한다.

## 규칙

1. **중요한 정보 포함**: 사용자의 주요 요청 사항, 질문, 문제 해결 포인트 등 중요한 내용만 포함한다.
2. **간결한 문장 사용**: 핵심적인 내용을 **간결하고 직설적으로** 표현한다.
3. **명확한 구조**: 요약은 **단락별로 구분**하며, 중요한 포인트를 쉽게 구별할 수 있도록 한다.
4. **JSON 제거**: 요청에 포함된 **JSON** 데이터나 코드 예시를 **평문**으로 변환하여 포함한다.
5. **상세하지 않게 요약**: 사용자가 명시적으로 요청하지 않은 세부 사항이나 기술적인 부분은 **요약에서 제외**한다.

## 예시

### 원본 요청

"AIMessage 클래스에서 content 필드에 여러 항목을 추가하고 싶은데, 각 항목은 tuple 형태로 되어있고, pydantic에서 이 데이터를 검증할 때 오류가 발생합니다. 오류 메시지는 여러 값들이 string 또는 dict로 되어야 한다고 말합니다. 이 문제를 해결하려면 어떻게 해야 할까요?"

### 요약된 내용 (Plain Text)

사용자는 AIMessage 클래스의 content 필드에 여러 항목을 추가하려고 하고, 각 항목이 tuple 형식으로 되어 있어 pydantic에서 오류가 발생하고 있습니다. 오류 메시지는 각 항목이 string 또는 dict 형식이어야 한다고 지적합니다. 해결책은 tuple을 제거하고, 각 항목을 string 또는 dict로 변경하는 것입니다.

## 추가 지침

-   내용은 **사용자의 요청 사항에 따라 달라질 수 있으므로**, 상황에 맞게 요약의 포커스를 조정할 수 있습니다.
-   **다시 말하기 방식**을 사용하여 핵심 내용을 정확히 전달하도록 한다.

-   내용에 흐름을 잡을 수 있는 포인트들이 있어야한다.

-   해당 내용은 이후에 참고 자료로 사용되는 용도이다.
